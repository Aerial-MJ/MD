### python

`python xxx.py > log` 是在命令行中运行 Python 脚本的一种方式。这里的含义是：

- `python xxx.py`：运行 `xxx.py` 这个 Python 脚本。
- `>`：这是一个重定向符号，它将标准输出（通常是打印到控制台的输出）重定向到指定的文件。
- `log`：这个文件名，表示将输出保存到名为 `log` 的文件中。

具体来说，命令 `python xxx.py > log` 会做以下事情：

1. 执行 `xxx.py` 这个 Python 脚本。
2. 将脚本的标准输出（即使用 `print()` 函数输出的内容）保存到 `log` 文件中，而不是显示在终端上。

如果 `log` 文件已经存在，它会被覆盖。如果 `log` 文件不存在，会创建一个新的文件。

如果你想追加输出内容到文件中而不是覆盖，可以使用 `>>`，例如：

```
python xxx.py >> log
```

这会把 `xxx.py` 的输出追加到 `log` 文件的末尾。

### nohup

`nohup` 命令的全称是 "no hang up"，它使得你运行的进程在你退出终端（关闭终端会话）后继续运行。结合重定向符号使用时，它的语法是：

```
nohup python xxx.py > log 2>&1 &
```

这个命令的各个部分含义如下：

1. **`nohup`**：使得程序在后台运行，并且不因关闭终端而终止。
2. **`python xxx.py`**：执行 `xxx.py` 脚本。
3. **`>`**：将标准输出重定向到 `log` 文件。
4. **`2>&1`**：将标准错误输出（`stderr`，即错误信息）也重定向到标准输出（`stdout`），即将错误信息也输出到 `log` 文件中。`2` 表示标准错误，`1` 表示标准输出，`&` 表示重定向到同一个目标。
5. **`&`**：将该命令放到后台运行，这样你可以继续在终端执行其他命令。

这个命令做了什么：

- 使用 `nohup` 保证脚本会在后台运行，即使你退出终端，脚本仍然会继续运行。
- 标准输出和标准错误输出都被重定向到 `log` 文件，这样你可以通过查看 `log` 文件来查看脚本的输出和错误信息。
- 使用 `&` 将脚本放到后台运行，不会阻塞你的终端。

如果你想让 `log` 文件记录更多的输出，也可以使用追加模式 `>>`：

```
nohup python xxx.py >> log 2>&1 &
```

这样，后续的输出会追加到 `log` 文件中，而不会覆盖之前的内容。
