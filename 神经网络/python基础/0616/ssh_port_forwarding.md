# SSH 端口转发详解

## 一、普通 SSH 是什么

最简单的场景：你有一台远程服务器（物理机或虚拟机）

```
Mac  ──SSH──►  远程服务器（物理机）
                  你可以在里面敲命令
```

```bash
ssh user@192.168.1.100    # 连上去，得到一个远程终端
```

---

## 二、Pod 是什么

Pod 是 **Kubernetes（K8s）里的一个容器**，可以理解为：

```
物理机（宿主机）
├── Pod A（一个独立的小"虚拟机"，跑你的训练任务）
├── Pod B
└── Pod C
```

**Pod 的特点：**
- 有自己独立的文件系统、网络、进程
- **没有固定 IP 对外暴露**，外部无法直接访问
- 只通过宿主机的某个端口"转发"进来（比如宿主机的 8416 端口映射到 Pod 的 22 端口）
- Pod 随时可能被销毁重建

所以访问路径是：

```
Mac  ──►  宿主机 10.164.10.177:8416  ──►  Pod 内部（你的训练环境）
```

宿主机的 8416 端口 = Pod 的 SSH 入口，K8s 帮你做了这个映射。

---

## 三、~/.ssh/config 配置文件

配置好别名之后，不需要每次手敲 IP、端口、用户名：

```
Host codelab-617710744          # SSH 别名，执行 ssh codelab-617710744 时使用
    HostName 10.164.10.177      # 服务器真实 IP
    Port 8416                   # SSH 端口（不是默认的 22）
    User hadoop-grocery-rc      # 登录用户名
    IdentityFile ~/.ssh/codelab_prod   # 私钥文件路径
    IdentitiesOnly yes          # 只用这个私钥，不尝试其他
    ForwardAgent yes            # 转发本地 SSH-agent，可以在远端使用本地的 key
```

配置好之后，`ssh codelab-617710744` 等价于：

```bash
ssh -p 8416 -i ~/.ssh/codelab_prod hadoop-grocery-rc@10.164.10.177
```

## 四、端口转发 vs 普通 SSH

| | 普通 SSH | SSH 端口转发 |
|--|---------|------------|
| 目的 | 获得远程终端，敲命令 | 把 Pod 里某个服务的端口"搬"到本地访问 |
| 命令 | `ssh codelab-617710744` | `ssh -L 6006:localhost:6006 -N codelab-617710744` |
| 结果 | 进入 Pod，可以敲命令 | 不进入 Pod，只建一条隧道 |

## 五、SSH 本地端口转发

```bash
ssh -L 本地端口:目标地址:目标端口 -N SSH别名
```

### 参数说明

| 参数 | 含义 |
|------|------|
| `-L` | 本地端口转发（Local forwarding），**定义转发规则**，建立隧道 |
| 本地端口 | 你 Mac 上监听的端口 |
| 目标地址 | 从远端（Pod）看过去的目标地址，`localhost` = Pod 自己 |
| 目标端口 | Pod 上要转发到的端口 |
| `-N` | **控制行为**，不执行远程命令，不打开 shell，只保持连接挂着（无任何输出，正常现象） |

### `-L` 三段参数的执行地点不同

```
-L   6006  :  localhost  :  6006
      ↑            ↑          ↑
   在Mac上      在Pod上      在Pod上
   生效          解析         生效
```

- **第一个 `6006`**：Mac 负责，Mac 开一个 6006 端口监听
- **`localhost:6006`**：Pod 负责，Pod 收到请求后自己去访问这个地址

### 实际例子

```bash
ssh -L 6006:localhost:6006 -N codelab-617710744
```

效果：本机访问 `http://localhost:6006` → 实际访问 Pod 内部的 6006 端口（TensorBoard）

## 六、为什么 localhost 是 Pod 而不是 Mac？

> **命令虽然在 Mac 上敲，但 `localhost` 是给 Pod 看的指令。**

SSH 协议的设计：`-L` 的"目标地址"部分，SSH 会把 `localhost:6006` 这个字符串**原封不动传给 Pod**，由 Pod 来解析和执行"去访问"这个动作。

类比理解：你打电话给快递员（SSH）：
> "你到了北京（Pod）之后，去**你自己住的地方**（localhost）取 6 号包裹"

你说"你自己住的地方"——这个"自己"是快递员（Pod）自己，不是你（Mac）。

### 什么时候才需要写 IP？

当目标**不是 Pod 自己**的时候：

```bash
# TensorBoard 跑在集群里另一台机器上
ssh -L 6006:192.168.1.99:6006 -N codelab-617710744
#              ↑ 另一台机器的 IP，Pod 帮你去访问它
```

## 七、TensorBoard 访问完整流程

**第一步：** 服务器上启动 TensorBoard（已在跑则跳过）

```bash
tensorboard --logdir output/qwen3vl_2b_v4_lora/runs --host 0.0.0.0 --port 6006 &
```

**第二步：** 本机新开一个终端，执行端口转发（挂着不要关）

```bash
ssh -L 6006:localhost:6006 -N codelab-617710744
# 无任何输出，正常现象，说明隧道建立成功
```

**第三步：** 本机浏览器访问

```
http://localhost:6006
```

## 八、整体数据流向

```
┌─────────────────────────────────────────────────────┐
│  你的 Mac                                            │
│                                                     │
│  终端1: ssh -L 6006:localhost:6006 -N codelab-xxx   │
│         （挂着，建隧道用）                            │
│                                                     │
│  浏览器: http://localhost:6006                       │
│         ↓                                           │
│         Mac 的 SSH 客户端收到请求                    │
└──────────────────┬──────────────────────────────────┘
                   │  SSH 加密隧道
                   ▼
┌─────────────────────────────────────────────────────┐
│  宿主机 10.164.10.177:8416                           │
└──────────────────┬──────────────────────────────────┘
                   │  K8s 端口映射
                   ▼
┌─────────────────────────────────────────────────────┐
│  Pod（你的训练容器）                                  │
│                                                     │
│  localhost:6006  ← TensorBoard 跑在这里              │
└─────────────────────────────────────────────────────┘
```

## 九、常见问题

### 端口转发命令挂着没有任何输出，正常吗？
正常。`-N` 就是让它不输出任何内容，安静地保持隧道连接。关掉这个终端，隧道就断了。

### 可以不加 `-N` 吗？
可以。不加 `-N` 的话，隧道照样建立，同时还给你一个远程终端（可以在 Pod 里敲命令）：

```bash
# 有 -N：只建隧道，终端挂着不动
ssh -L 6006:localhost:6006 -N codelab-617710744

# 没有 -N：建隧道 + 打开远程 shell
ssh -L 6006:localhost:6006 codelab-617710744
```

### 可以写 Pod 的 IP 代替 localhost 吗？
可以，效果一样，只是多此一举：

```bash
ssh -L 6006:10.164.10.177:6006 -N codelab-617710744  # 等价，但绕了一圈
ssh -L 6006:localhost:6006 -N codelab-617710744       # 推荐写法
```
