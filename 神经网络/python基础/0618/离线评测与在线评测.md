# 离线评测 vs 在线评测 vs A/B 实验 vs 灰度测试

## 1. 离线评测 vs 在线评测

### 1.1 离线评测（Offline Evaluation）

**定义**：用一批提前准备好的、固定的标注数据集来跑模型，得到指标数字，不涉及真实线上流量。

**典型做法**：

- 用 `ig_sft_v4_test.json`（1000 条）跑各模型
- 得到准确率、P/R/F1 等数字
- 整个过程在本地离线完成，不影响线上

**离线评测的问题**：

- 测试集和训练集来自同一批数据（虽然无图片重叠，但场景分布、图片类型、标注风格完全一致）
- 测试集 GT 来自 Qwen3-32B-VL 自动标注 → 模型在"自己老师出的题"上考试，天然偏高
- 无法反映真实线上图片的分布（长尾噪声、异常图片、新场景等）

### 1.2 在线评测（Online Evaluation）

**定义**：把模型部署到线上，用真实业务流量来评测效果，通常需要人工二次核查或 A/B 实验。

**常见做法**：

| 方式 | 说明 |
|------|------|
| Shadow 模式 | 模型跑线上流量但不影响决策，人工抽样核查预测结果 |
| A/B 实验 | 一部分流量走新模型，一部分走老逻辑，对比业务指标（误拦截率、漏判率等） |
| 人工审核抽检 | 线上预测结果定期抽样，人工判断对错，统计真实准确率 |

**优势**：
- 测的是真实分布，不存在训练/测试同源问题
- 能发现离线测不到的问题（如新出现的无效图类型、图片质量差异等）

### 1.3 结合项目的数据流对比

```
当前状态（离线）
  训练集 ──┐
           ├── 同一批 Qwen3-32B-VL 标注数据 → 测试集 GT 可信度存疑
  测试集 ──┘

更严格的离线评测
  完全隔离的新图片 → 人工独立标注 GT → 离线跑模型
  （时间段不同 / 来源系统不同 / 标注人不同）

在线评测（最终验收）
  线上真实流量 → Shadow 跑模型 → 人工抽检 → 统计业务口径误判率/漏判率
```

> 💡 **简单说**：离线评测是"考试题自己出的"，快、成本低，但可能虚高；在线评测是"考真实试卷"，慢、成本高，但是真实水平的最终验证。

## 2. A/B 实验 vs 灰度测试

### 2.1 两者关系

> **灰度是 A/B 实验的基础设施，A/B 实验是灰度的一种目的。**

### 2.2 灰度测试（Grayscale / Canary Release）

**本质**：一种**发布策略**，核心目的是**控制风险**。

把新版本只推给一小部分流量（如 5%、10%），观察有没有崩溃、报错、性能问题，没问题再逐步扩大到 100%。

```
全量流量
  ├── 5%  → 新模型（灰度）
  └── 95% → 老逻辑（兜底）
```

- 关注的问题：新版本**稳不稳**、会不会出故障 → **工程稳定性视角**

### 2.3 A/B 实验（A/B Test）

**本质**：一种**科学对比方法**，核心目的是**量化效果差异**。

同时运行两个版本，通过统计方法判断哪个更好，关注业务指标的提升。

```
全量流量
  ├── 50% → A 组（老逻辑/对照组）
  └── 50% → B 组（新模型/实验组）
         ↓
  对比：误判率、漏判率、审核通过率...
  统计显著性检验 → 得出结论
```

- 关注的问题：新版本**比老版本好多少** → **业务效果视角**

### 2.4 对比总结

| 维度 | 灰度测试 | A/B 实验 |
|------|----------|----------|
| **核心目的** | 控制发布风险 | 量化效果差异 |
| **流量分配** | 小比例（5%~20%） | 通常各 50% |
| **持续时间** | 短，确认稳定即扩量 | 较长，需积累足够样本量 |
| **关注指标** | 系统稳定性、错误率 | 业务 KPI（误判率、精准率等） |
| **统计要求** | 无需显著性检验 | 需要统计显著性 |

## 3. 完整上线流程

实际工程中，一个完整的上线流程通常是：

```
离线评测（当前阶段）
    ↓
小流量灰度（5% Shadow，人工抽检，验证稳定性）
    ↓
A/B 实验（新模型 vs 老规则，对比业务误判率）
    ↓
全量上线
```

### 3.1 结合图片审核模型的建议路径

1. **灰度阶段**：先让新模型以 Shadow 模式跑 5% 流量，只记录结果不影响决策，人工抽查看有没有明显离谱的误判

2. **A/B 阶段**：确认稳定后，拿新模型（B 组）对比当前线上规则/DINOv2（A 组），统计**无效图漏判率**和**有效图误拦截率**这两个核心业务指标的差异

3. **全量**：A/B 结果显著且 B 组更优，扩量上线

> 💡 这样汇报时的数字就从"离线 99.9%"变成了"线上 A/B 实验中误判率降低 X%"，说服力会强很多。
